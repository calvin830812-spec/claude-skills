# 寶舖建設集團｜開發部每日工作日報

當使用者說「填日報」、「今天做了什麼」、「寫日報」，立即開始以下流程。
目標：**至少 2 輪確認**，確保每件事符合人事時並形成閉環。

---

## 核心原則：人事時 × 閉環

每件事在產出前，必須通過以下兩項檢查。

### 人事時檢查

| 項目 | 定義 | 合格標準 |
|------|------|---------|
| **事** | 今天這件事的結論或結果是什麼 | 有具體結論，不只描述行動（「開了會」「整理了資料」不合格） |
| **人** | 需要誰配合 | 有填配合對象，或明確確認是獨立作業 |
| **時** | 何時前完成 + 進度狀態 | 有時間點，進度已勾選（完成／進行中／延後） |

### 閉環檢查

今日事項與明日事項之間必須形成連貫：

| 今日狀態 | 閉環要求 |
|---------|---------|
| 進行中 | 明日三件事中必須有接續此事項的安排 |
| 延後 | 明日三件事中必須有接續，且需說明延後原因與新時間 |
| 完成但帶出後續行動 | 明日三件事中需安排對應的下一步 |

---

## 流程

### 第 1 輪：接收原始輸入

說：
> 「直接貼就好，不用整理格式：
> 姓名、日期、今天做了什麼、明天要做什麼。」

接收使用者的原始內容後，**不要立刻追問**，先完整解析。

---

### 第 2 輪：人事時確認（必做，不可跳過）

**無論輸入是否完整，都必須執行此輪。** 對照人事時檢查，有缺漏就追問，沒缺漏就列出解析結果請使用者確認：

有缺漏時，**把所有缺漏合併成一次追問**：

| 缺漏 | 追問 |
|------|------|
| 今日只描述行動，無結論 | 「X 件事的結論或結果是什麼？」 |
| 看不出進度狀態 | 「X 件事是完成、進行中，還是延後？」 |
| 進行中但未說目前到哪 | 「X 件事目前做到哪了？還剩什麼？」 |
| 延後但未說新時間 | 「X 件事延到什麼時候？原因是什麼？」 |
| 明日目標只說要做什麼，無目標狀態 | 「X 件事明天做完後，要達到什麼具體結果？」 |
| 缺少配合對象 | 「X 件事需要誰配合？還是獨立作業？」 |
| 缺少時間點 | 「X 件事何時前完成？」 |

無缺漏時，列出解析摘要請確認，例如：
> 「我這樣理解，確認一下：
> 第1件：[結論]，配合：[人]，期限：[時]，進度：完成
> 第2件：…
> 第3件：…
> 這樣對嗎？」

代補欄位標示「（代補）」。等使用者回覆後再進入第3輪。

---

### 第 3 輪：閉環檢查（必做，不可跳過）

**無論今明是否明顯連貫，都必須執行此輪。** 人事時確認完畢後，逐項核對閉環，說明結論：

有缺口時：
> 「確認一下今明的連貫性：
> X 件事今天進行中，明天的計畫裡有接續嗎？
> Y 件事今天完成後，下一步是 Z，明天有安排嗎？」

無缺口時，仍需明確說出核對結論，例如：
> 「今明連貫性確認：三件事的今明接續都有安排，閉環沒問題。」

再呈現完整草稿。

---

### 呈現草稿，等待確認

格式如下：

```
【今日三件事】

第1件事（最重要）｜專案：___
今日結果：___
配合：___　期限：___　進度：___
需主管知道：___（無則略）
進度說明：___（進行中／延後才填）

第2件事 … 第3件事 …

【明日三件事】

第1件事（最重要）｜專案：___
明日目標：___
配合：___　期限：___

第2件事 … 第3件事 …

（代補欄位：___，請確認）
```

使用者若回覆「可以」或指定修改，**立即處理修改並執行出檔**，不再追問。

將資料填入 `generate_report.py` 頂部變數區後執行：

```bash
python3 generate_report.py
```

告知存檔路徑。

---

## 執行原則

- 先接受原始輸入，不打斷
- 人事時缺漏合併成一次追問（第 2 輪）
- 閉環缺口單獨一次追問（第 3 輪）
- 代補內容明確標示，讓使用者確認
- 使用者確認「可以」後才出檔，不提前產出

## 範例對話

```
AI：直接貼就好，不用整理格式：
    姓名、日期、今天做了什麼、明天要做什麼。

同事：莊穎霖，115/5/11。
      今天：羅斯795比較元創兆碩兩方案，元創B6反計容積有問題，兆碩B4解決了，
      完成三筆提案資料。信義364約到住戶明天碰面。復興623整理完投報表進行中。
      明天：羅斯795送董事長確認後交五福主委。信義364拜訪住戶爭取同意書。
      復興623早上跟俞安交接。

【第2輪：補人事時】

AI：補幾個資訊：
    復興623今日進度——做到哪了？何時完成？
    信義364明日——拜訪完後要達到什麼具體結果？
    三件事的時間點都沒填，分別是何時前完成？

同事：復興623論述框架完成，待都更顧問做簡報，今天完成。
      信義364希望拿到都更同意書，或至少釐清對方的顧慮。
      三件都是今天完成。

【第3輪：閉環檢查】

AI：確認今明連貫性：
    復興623今天完成，明天的交接是承接什麼內容？
    信義364今天確認見面，明天拜訪後若沒拿到同意書，後續怎麼處理？

同事：復興623交接住戶聯絡資料和簽署狀況。
      信義364若拿不到，至少釐清對方顧慮，下次再約。

AI：（呈現完整草稿）
    這樣可以嗎？

同事：可以。

AI：✅ 日報已產出：output/1150511-開發部-日報-莊穎霖.docx
```
